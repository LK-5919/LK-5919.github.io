1.meta标签的使用

​ 元数据（metadata）；

​ 首先明确meta标签中提供关于HTML文档的元数据；元素据不会在页面显示，但是它里面的元数据对机器是可读的；meta标签常用于规定对页面的描述，关键词，文档作者等；

​ 元数据可用于浏览器（如何显示页面charset，重现加载页面http-equiv=”Refresh”），也可以用于搜索引擎（keyword）等；

​ meta的简单使用

​ 1.当meta标签里有http-equiv或name属性时，一定要有content属性对其进行说明；

<meta name = "keywords" content = "这里的关键字会被搜索引擎所使用">

http-equiv 属性可以添加一些自定义的或额外添加的http头部内容，例如不想使用js来实现页面的重定向的话可以在http头部内容中添加如下属性来控制：
<meta http-equiv="Refresh" content = "3;url=http://...."/>
content里面的3是指定3秒后跳转到指定页面；

​ 2.name属性是比较常使用的，可以在这里指定浏览器如何解析或者一些浏览器兼容问题；注意：name属性中的值是有规定的，不是自定义的，否则是没有意义的；

​ 常用的mate标签总结

<meta charset="utf-8">
<!-- 页面关键词 keywords -->
<meta name="keywords" content="your keywords">
<!-- 页面描述内容 description -->
<meta name="description" content="your description">
<!-- 定义网页作者 author -->
<meta name="author" content="author,email address">

移动端页面布局元数据声明
<meta name="viewport" content="width=device-width, initial-scale=1.0">

2.input 标签的file属性

​ 首先明确input标签是基于Web的表单创建交互式控件，以便接受来自用户的数据；可以通过使用type属性来决定输入的各自类型；

​ 使用type = “file” 可使得用户通过input元素选择一个或多个文件（需要multiple属性）以提交表单的方式上传到服务器。

​ 当使用type = “file” 属性的时候，它有以下几个附加属性：

​ 1.accept：一个或多个唯一文件类型的说明符，描述允许的文件类型，它的值为字符串，定义input应该接受的文件类型；

<input type="file" id="docpicker"
  accept=".doc,.docx,application/msword,application/vnd.openxmlformats-officedocument.wordprocessingml.document">

​ 2.capture：捕获图像或视频数据的源（前提是accept属性中指定的是图片或者视频类型），值 user 表示应该使用前置摄像头和/或麦克风。值 environment 表示应该使用后置摄像头和/或麦克风。

​ 3.files：FileList 对象每个已选择的文件。如果 multiple 属性没有指定，则这个列表只有一个成员。

​ 4.multiple：布尔值，如果出现，则表示用户可以选择多个文件。

5.v-if 和 v-show 区别

​ 实现原理：v-if是动态的向DOM树内添加或删除DOM元素；v-show是通过设置DOM元素的display样式属性来控制元素的显示与隐藏；

​ 编译过程：v-if切换有一个局部编译/卸载的过程，切换过程中会进行销毁和重建内部的事件监听和子组件；v-show只是简单的基于CSS切换；

​ 编译条件：v-if是惰性的，如果初始条件为false，则什么也不做，只有当条件变为true时才开始局部编译；而v-show即使是在初始条件为false时也会编译缓存，而且保留DOM元素；

​ 性能消耗：v-if有更高的切换消耗；v-show有更高的初始渲染消耗；

​ 使用场景：v-if适合条件不大可能改变的；v-show适合频繁切换；

6.link的hrfe和script的src区别

​ 首先明确src和href都是用来引用外部的资源；这两者的区别如下：

​ href：全称：Hypertext Reference，超文本引用，它指向一些网络资源，建立和当前元素（a标签）或者说是当前文档（link）的链接关系。在加载它的时候，不会停止对当前文档的处理，浏览器会继续往下执行。

​ src：全称：source，资源，表示对资源的引用，它指向的内容会嵌入到当前标签（img，iframe，video，audio，script）所在的位置（这种标签称为可替换元素）。src会将它所指向的资源下载到当前文档内，如请求js脚本。当浏览器解析到该元素时，会暂停其他资源的下载和处理，直到将src所指向的资源加载，编译，执行完毕。（所以实际开发中把js脚本写在页面底部比较稳妥）；

​ 总结：href用于表示引用资源与当前元素或文档的关系，src会替换掉当前的标签；遇到href所指向的引用资源时页面会并行加载不停止对当前文档的处理，但是src所指向的资源需要浏览器暂停当前对文档的处理来加载当前src的内容，加载完成后才会继续往下继续对文档的处理；

​ 补充：可替换元素与非替换元素

​ 可替换元素（replaced element），它所展现的效果不由CSS来控制，这些元素是一种外部对象，他们外观展示的渲染是独立于CSS；可替换元素一般也具有自己的宽度，高度，浏览器会根据元素的标签和属性来决定具体显示的内容。常见的可替换元素，例如

<iframe>、<video>、<embed>、<img>

​ 非替换元素（non-replaced element），它的内容是包含在文档中的元素，它的内容受CSS渲染控制，非替换元素的内容不会超过CSS的模型范围，CSS在渲染时会考虑到非替换元素的内容。常见的非替换元素有

<div>,<p>,<h1>~<h6>,<table>

7.animation和transition

​ animation是动画属性，它的实现不需要通过事件触发，设定好时间之后可以自己执行，它可以设置多个关键帧（@keyframe）完成动画；

​ transition是过渡属性，强调过渡，它的实现需要通过事件的触发才会开始执行，它需要设置一个开始关键帧和结束关键帧。

​ 区别：

​ animation可以设置很多的属性，比如循环次数，动画的结束状态等等，但是transition触发一次才才执行一次；

​ animation可以通过@keyframe设置每一帧（即有多个状态），但是transition只有开始和结束两帧；

​ transition有一定的限制，并不是所有的CSS属性都具有过渡效果；

10.vue-router中使用的路由跳转（push，go，replace方法的区别）

​ this.$router.push()方法在跳转到指定的url时，会向history栈添加一个记录，点击浏览器的后退按钮时可以回退到上一个页面；

​ this.$router.replace()方法在跳转到指定页面的时候会把history栈的栈顶url给替换，点击浏览器的回退按钮时不能回到上一页面，只会回退到上上页面（如果存在的话）；

​ this.$router.go()方法可以传入一个数值，指定网页向前或向后跳转多少个页面，类似window.history.go(n)，n可为整数或者负数；

11.操作dom节点（querySelector与getElementById的区别）

​ 首先明确这两个方法都是用来查询dom节点的；

​ querySelector()接受的参数是一个css选择器名，它会返回在网页中第一个符合该css选择器名条件的元素节点；

​ getElementById()方法接受的参数是网页中元素节点的id属性名，它将返回网页中第一个符合该id属性的元素节点；

​ querySelectorAll()方法接受的参数是CSS选择器，它返回文档中匹配的CSS选择器中的所有元素节点列表；（数组，即使只查询到一个元素节点也是返回数组）；

​ getElementsByName()：返回带有指定名称name的对象的集合。返回的是元素的数组，而不是一个元素（由于name不唯一）；

​ getElementsByTagName()：返回带有指定标签名的对象集合；

12.axios拦截器

​ 可以给axios配置请求拦截器和响应拦截器，用于处理一些请求前和响应后的业务；

//引入axios
import axios from 'axios';
//引入router
import router from '@/router';

//创建一个axios实例,需要传入一个基本的配置对象
let myAxios = axios.create({
    baseURL:'api/',
    timeout: 15000 //毫秒
}) 

//添加请求拦截器，在请求拦截器中加入token
myAxios.interceptors.request.use(function(config){
    //config中的headers属性中可以自定义我们需要发送的请求头
    
    //查询当前是否存在token，如果存在，在请求前将其加入请求同一同发送给服务器
    if(sessionStorage.getItem('token')){
        //给请求头添加Authorization属性，值为token
        config.headers.Authorization = sessionStorage.getItem('token');
    }
    return config;
},(error) => {
    return Promsie.reject(error);
});

//添加响应拦截器，当登录失效时强制跳转到登录页面
myAxios.interceptors.response.use((res) => {
    //请求成功
    return res;
}, (error) => {
    //请求失败
    if(error.response && error.response.status === 401){
        //登录失效，移除token
        sessionStorage.removeItem('token');
        //跳转到登录页面
        router.push('/login');
    }
    return Promise.reject(error);
});

//将加工后的axios暴露出去
export default myAxios;

5.Promise的race()和all()方法

​ 首先明确race和all都是将多个Promise封装成一个新的Promise；

​ race()方法执行结果取决宇第一个Promise的返回结果，reject执行catch，resolve执行then；

​ all()方法所有Promise的结果都返回resolve才会执行then，返回结果为一个存放所有结果的数组里，如果有任何一个返回reject，就会执行catch；（使用all方法时，当里面有一个Promise返回了reject，那就会直接执行catch，而不会等其他的Promise）；

6.async和await到底是什么

​ Promise -> Generator -> async/await

​ 以下以三级联动获取省市县的需求来逐步探讨：

在还没提出Promise的时候，我们如果实现上面的需求，大致代码如下：

//使用jQuery的Ajax请求数据
function getCity(){
    //获取省数据
    $.ajax({
        type:'GET',
        url:'getProvince',
        dataType:'json/jsonp',
        success:function(res){
            //请求省数据成功
            console.log('省的数据');
            //拿到省的数据后去请求市的数据
            $.ajax({
                ...
                success:function(res){
                    //请求市的数据成功
                    console.log("市的数据");
                    //拿到市的数据后去请求县的数据
                    $.ajax({
                        ...
                        success:function(res){
                            //请求县的数据成功
                            console.log("县的数据");
                            ...
                        }
                    })
                }
            })
        }
    })
}
        
从上面可以看到，在ajax请求成功的回调中再发送异步请求，回调中嵌套回调，像这种情况就称为回调地狱，代码会往水平方向扩展，不利于代码的阅读和维护；
        
我们使用Promise来解决上面的需求
 new Promise(function(resolve,reject){
        ...
        //请求省的数据成功
        resolve("省的数据");
    }).then((res) => {
        //拿到省的数据后去请求市的数据
        return new Promise((resolve,reject)=>{
            ...
            //请求市的数据成功
            resolve("市的数据");
        })
    }).then((res)=>{
        //拿到市的数据以后请求县的数据
        return new Promise((resolve,reject)=>{
            ...
            //请求县的数据成功
            resolve("县的数据");
        })
    }).then((res)=>{
        //展示三级联动数据
        console.log(res);
    }).catch((error)=>{
        console.error(error);
    });
    
    从上面的代码可以看出，Promise的链式编程解决了回调地狱的问题，它可以将异步代码以同步代码的方式书写，虽然Promise解决了回调地狱的问题，但是它本身仍然存在回调问题；所以由衍生出generator函数，需要搭配yield使用，每次执行都需要调用next()方法；

        let county = yield getCounty(city);
        return {province,city,county};
    }

    let iterator = getThreeInfo();
    iterator.next();//这里将执行generator函数里的第一个yield关键字后面的表达式
    
    //定义一个请求省数据的函数
    function getProvince(){
        ...

        $.ajax({
            ...
            success:function(res){
                //调用next方法，并将省的数据，作为第一个yield关键字后面的返回值
                iterator.next(res);
            }
        })
    }
    
    //定义一个请求市数据的函数
    function getCity(province){
        ...
        //请求市的数据
        $.ajax({
            ...
            success:function(res){
                //调用next方法，并将市的数据传入，作为第二个yield关键字后面的返回值

            }
        })
    }

        ...

            ...
            success:function(res){

         let county = await new Promise((resolve,reject)=>{
            ...
            //请求县的数据成功
            resolve("县的数据");
        });
        //await后面等待一个Promise，如果await还没执行完，它后面的代码也会被堵塞
         return Promise.resolve({province,city,county});
        }catch(e){
            return Promise.reject(error);
        }
    }
    getThreeInfo().then((value)=>{
        console.log(value);
    },(error)=>{
        console.error(error);
    });
    
    从上面的代码就可以知道，async函数其实就是generator函数+自执行

计网部分

1.http和https区别（https的s是什么）

​ HTTP协议：Hyper Text Transfer Protocol（超文本传输协议），属于应用层协议，基于TCP协议（TCP协议是端到端的可靠连接协议）的应用层传输协议。HTTP协议是用于本地浏览器和服务器传输超文本的传输协议。它是一种无状态协议；

​ 在平时的从浏览器中键入URL请求到网页加载完成所经历的过程：

​ 1.对输入的url地址进行DNS域名解析，得到对应的IP地址；

​ 2.向IP地址服务器发起TCP请求，经过三次握手后建立TCP连接；

​ 3.向服务器发起HTTP请求；

​ 4.服务器接受到HTTP请求，处理请求，响应HTTP请求；

​ 5.浏览器根据响应的数据解析html代码，并请求html代码中的资源（js脚本，css，图片等）；

​ 6.浏览器渲染页面并显示（页面渲染步骤：先解析html文件生成DOM树，解析CSS文件构成渲染树，边解析边渲染）；

​ HTTP请求的组成结构：

​ （1）请求行（包含请求方法，请求url，http协议版本，换行）

​ GET/example.html HTTP/1.1(CRLF)

​ 请求方法有如下几种：

​ GET：请求获取URL所标识的资源

​ POST：在请求URL所标识的资源后增加新的数据

​ PUT：请求服务器存储一个或修改一个资源，使用URL作为其标识

​ DELETE：请求服务器删除URL所标识的资源

​ HEAD：请求获取URL所标识的资源的响应消息报头

​ TRACE：请求服务器回送收到的请求信息，主要用于测试或诊断

​ OPTIONS：请求查询服务器性能或查询与资源相关的选项和需求

​ CONNECT：保留将来使用

​ （2）请求头（消息报头）

​ 请求头是由一些键值对组成，每行一对关键字和值用 : 分隔。允许客户端向服务器发送一些附加信息或者客户端自身的信息。常见的请求头如下：

​ （3）空行（CRLF）

​ 最后一个请求头之后是一个空行，发送回车符和换行符，通知服务器以下不再有请求头。

​ （4）请求正文（请求体）

​ GET方法中请求体是空的，请求体多在POST方法中使用

​ HTTP响应的组成结构：

​ HTTP响应由3部分组成：

​ （1）状态行

​ 状态行包含三部分，HTTP协议的版本号，状态码（用来表示网页服务器响应状态的3位数字代码，1xx-5xx），以及对状态码的文本描述。

​ （2）消息报头

​ （3）响应正文

​ HTTP协议的特点：无连接（每次只处理一个请求，服务器处理完客户的请求后并收到应答后即断开连接）；无状态（HTTP协议无法根据之前的状态进行本次的请求处理）；灵活（HTTP允许传输任意的数据类型在传输的类型由Content-Type加以标记）

​ HTTP协议的缺点：

​ （1）通信使用明文不加密；

​ （2）不验证通信方身份，可能遇到伪装；

​ （3）无法验证报文的完整性，可能已遭篡改；

​ 总的来说：不安全！

​ HTTPS协议

​ https协议就是身披SSL外壳的Http协议。https协议是一种通过计算机网络进行安全通信的传输协议，经由http进行通信，利用SSL／TLS进行加密数据包。https默认的端口号是443。HTTP+加密+认证+完整性保护=HTTPS

​ 简述SSL/TLS握手过程

​ （1）客户端向服务端发送Client Hello打招呼（其中包括客户端支持的TLS版本和选择的加密套件，并生成第一随机数一同发送）

​ （2）服务端向客户端发送Serve Hello（，开始服务器打招呼其中包括服务端选择的TLS版本，加密套件，并生成第二随机数一同发送）

​ （3）紧接着服务端再发送Certificate（服务端的SSL证书）

​ （4）服务器再发送Serve Key Exchange（服务端的公钥）

​ （5）服务端发送Serve Hello Done（服务器打招呼结束）

​ （6）客户端再生成一个第三随机数（也称为预主密钥），客户端使用之前接受的公钥加密刚才生成的预主密钥，并向服务端发送Client Key Exchange，Change Ciper Spec，Encrypted Handshake Message（其中包含加密后的预主密钥一同发送）

​ （7）服务端接受到加密后的预主密钥以后，使用服务端的私钥进行解密，获得未加密的预主密钥（第三随机数）

​ （8）客户端和服务端都使用第一随机数，第二随机数，预主密钥计算出会话密钥（即客户端和服务端会得到同一个会话密钥）

​ （9）此时服务端可客户端都拥有了同一个会话密钥，之后的会话都使用这个会话密钥对数据进行加密，即后面使用的是对称加密

​ HTTPS的特点

​ 内容加密（传输的数据不是明文），验证身份（通过证书认证客户端访问的是目标服务器），保证数据的完整性（防止传输的数据遭到修改）

​ HTTPS的缺点

​ HTTPS协议的握手阶段比较耗时，会使页面加载时间延长；SSL证书需要绑定IP，不能再一个IP上绑定多个域名；

2.GET和POST请求的区别（能干什么）

​ 首先明确GET请求和POST请求都是HTTP协议中的两中方法。

​ 1.GET的数据在URL中是可见的，POST请求是不显示在URL中的（即GET参数通过url传递，POST放在请求体中）；

​ 2.GET请求对长度是有限制的，POST请求长度是无限制的；

​ 3.GET比POST不安全，参数会暴露在URL中；

​ 4.GET请求浏览器会主动缓存，但是POST请求不会；

​ 5.GET请求产生一个TCP数据包，POST请求会产生两个TCP数据包。GET请求中浏览器会把http header 和 data 一并发出，对于POST请求，浏览器会先发送header，服务器响应100 continue后，浏览器再发送data；
