ECMAScript介绍

ECMAScript 是由 Ecma 国际通过 ECMA-262 标准化的脚本程序设计语言。

let关键字

使用 let 关键字来声明变量；
	
let a = 1;  //声明Number类型基本变量
let b = "hello";  //声明String类型基本变量
let flag = true;  //声明Boolean类型基本变量
let obj = {name:"LK"}; //声明对象
let arr = [1,3]; //声明数组
let fun = function(){   //声明函数
    console.log("hello");
}

除了以下几点和 var 关键字不一样以外其他用法两者一致；

1.不允许重复声明变量（使用var关键字可以反复声明同一变量，js引擎在解析时发现用var关键字声明的变量已经存在时直接跳过而不会报错，而使用 let 关键字会报错）；
    
let a = 1;
let a = "hello";  //报错 Identifier 'a' has already been declared

var a = 2;
var a = "hello";  
注意 ： 已经使用let关键字声明的变量再次使用var关键字声明同样会报错！！！

2.具有块级作用域：（使用大括号{}包起来的属于一个块级作用域，块级作用域内声明的变量只在当前块级作用域下可见，for，if，while中的内容体都属于块级作用域）；

{
    let a = 1;
}
console.log(a);  //报错 a is not defined

if(true){
    let b = 2;
}
console.log(b);  //报错 b is not defined

if(true){
    var c = 1;
}
console.log(c);  // 输出 1

使用var关键字给按钮绑定事件
for(var j=0;j<btnList;j++){
    btnList[j].onclick = function(){
        console.log(i);
        //注意这里的this依旧是当前被点击的按钮对象
    }
}

/*
触发效果 ： 点击每一个按钮都是输出btnList的长度值；
原因分析：在for循环中使用 var关键字声明的变量将成为全局变量（不存在块级作用域和村存在变量声明提升），回调函数的执行是在初始化代码执行完毕以后，用dom事件模块监听到事件触发才把对应的函数回调放入循环事件队列中执行的，此时，i作为全局变量已经自增到了btnList的长度值，此时执行回调函数时输出的是全局变量变量的值；
以上代码具体展开：

var j; 
{j = 0; btnList[j].onclick = function(){console.log(j)};}
{j = 1; btnList[j].onclick = function(){console.log(j)};}
{j = 2; btnList[j].onclick = function(){console.log(j)};}
{j = 3;}

因为js都是先执行初始化代码，在执行回调代码，所以在绑定的function(){console.log(j)}; 语句都是在初始化代码之后，也就是这里的 j = 3 ;之后；
在执行回调时，回调函数输出访问的是全局的j，而不是当前块级作用域中的j（var 变量不具有块作用域）
*/

利用let的块级作用域绑定事件：
var  btnList = document.getElementsByTagName("btn");

for(let i =0;i<btnList.length;i++){
    btnList[i].onclick = function(){
        console.log(i);
    }
}

/*
触发效果：点击对象按钮就能正确输出它的索引值；
原因分析：在for循环中使用let关键字声明的变量具有块级作用域和不存在变量声明提升；每一次执行一次for循环都是一个全新的let i；循环之间的i具有块级作用域不会相互影响；
以上代码具体展开：

{let i = 0;bthList[i].onclick = function(){console.log(i)};}
{let i = 1;bthList[i].onclick = function(){console.log(i)};}
{let i = 2;bthList[i].onclick = function(){console.log(i)};}
{let i = 3;}

let 声明的关键字具有块级作用域，所以在执行回调函数时，输出的i都是访问当前块级作用域下的i；
*/

3.let 关键字声明的变量不存在变量提升；
console.log(a);  //报错 Cannot access 'a' before initialization
let a = 1;

console.log(b); //输出 undefined
var b = 1;

4.不影响作用域链
{
    let name = "LK";
    function fun(){
        console.log(name);
    }
    fun();  //输出 LK，在当前函数的块级作用域下没有查找到name，则前往外层作用域查找
}

 let school = '尚硅谷';
        {
            // let school = '尚硅谷';
            function fn(){
                console.log(school);
            }
            fn();  //输出 尚硅谷
        }

const关键字

使用const关键字声明 常量；
	1.const声明的变量在声明时就需要赋初值；
	2.const声明的基本数据类型值不能再改变，否则会报错；
    3.但是const声明的对象（数组，函数）里面的值可以改变，因为栈中保存的const变量是保存的对象地址的引用；
    4.const 声明的变量也具有块级作用域；
    5.不允许重复声明；
    
    应用场景：声明对象类型使用 const，非对象类型声明选择 let；
    
    const num;   //报错 Missing initializer in const declaration

    const a = 1;
	a = 2 ;  // 报错 Assignment to constant variable.

	const obj = {
        name:"Lk"
    }
    obj.name = "LBJ";
	console.log(obj);  // {name: 'LBJ'}

解构赋值

https://www.cnblogs.com/xiaohuochai/p/7243166.html

ES6 允许按照一定模式从数组和对象中提取值，对变量进行赋值，这被称为解构赋值。

注意使用解构赋值时，解构的类型和赋值的类型要一致，对象->对象 数组 ->数组

默认值：
　使用解构赋值表达式时，如果指定的局部变量名称在对象中不存在，那么这个局部变量会被赋值为undefined；
 
 let node = {
    type: "Identifier",
    name: "foo"
};
let { type, name, value } = node;
console.log(type); // "Identifier"
console.log(name); // "foo"
console.log(value); // undefined

　这段代码额外定义了一个局部变量value，然后尝试为它赋值，然而在node对象上，没有对应名称的属性值，所以像预期中的那样将它赋值为undefined

　　当指定的属性不存在时，可以随意定义一个默认值，在属性名称后添加一个等号(=)和相应的默认值即可

let node = {
    type: "Identifier",
    name: "foo"
};
let { type, name, value = true } = node;
console.log(type); // "Identifier"
console.log(name); // "foo"
console.log(value); // true

为非同名局部变量赋值：
　如果希望使用不同命名的局部变量来存储对象属性的值，ES6中的一个扩展语法可以满足需求，这个语法与完整的对象字面量属性初始化程序的很像；
 
 let node = {
    type: "Identifier",
    name: "foo"
};
let { type: localType, name: localName } = node;
console.log(localType); // "Identifier"
console.log(localName); // "foo"


//数组的解构赋值
const arr = ['LK','Zk','Lili'];
let [p1,p2,p3] = arr;
console.log(p1,p2,p3);  // 输出 LK ZK Lili

let {s1,s2,s3} = arr;
console.log(s1,s2,s3);  //输出 undefined undefined undefined （解构类型不一致）

//对象解构赋值
const obj = {
    name:"LK",
    age:12
}
let {name,age} = obj;
console.log(name,age);  //输出 LK 12

//复杂解构
let wangfei = {
 name: '王菲',
 age: 18,
 songs: ['红豆', '流年', '暧昧', '传奇'],
 history: [
 {name: '窦唯'},
 {name: '李亚鹏'},
 {name: '谢霆锋'}
 ]
};
let {songs: [one, two, three], history: [first, second, third]} = 
wangfei;

console.log(one,two,three,first,second,third);
//输出 红豆 流年 暧昧 {name: '窦唯'} {name: '李亚鹏'} {name: '谢霆锋'}

解构赋值一般用于获取对象的方法用于简便表示：
const obj = {
    name:"lk",
    fun:function(){
        console.log(this);
    }
}
let {fun} = obj; 
fun();  //输出 window

模板字符串

模板字符串（template string）是增强版的字符串，用反引号（``）标识，特点：
1) 字符串中可以出现换行符
2) 可以使用 ${xxx} 形式输出变量

const str = `hello`;
console.log(str);  // 输出 hello
console.log(typeof str);  //输出 string

let Text = `床前明月光
            疑是地上霜
            举头望明月
            低头思故乡`;
console.log(Text);

let num = 1;
console.log(`${num}2`);  //输出 12
console.log(typeof `${num}2`);  //输出 string

简化对象写法

ES6 允许在大括号里面，直接写入变量和函数，作为对象的属性和方法。这样的书写更加简洁。

let name = '尚硅谷';
let slogon = '永远追求行业更高标准';
let improve = function () {
 console.log('可以提高你的技能');
}
//属性和方法简写
let atguigu = {
 name,
 slogon,
 improve,
 change() {
 console.log('可以改变你')
 }
};
//对象中的属性如果和外部的全局变量名字一致，可以直接省去key:value 的写法，直接写value，对象中的方法也可以直接省去function关键字；

箭头函数

https://blog.csdn.net/qq_32614411/article/details/80897256
ES6 允许使用「箭头」（=>）定义函数。
箭头函数的注意点:
1) 如果形参只有一个，则小括号可以省略
2) 函数体如果只有一条语句，则花括号可以省略（省略花括号时return必须省略），函数的返回值为该条语句的执行结果
3) 箭头函数 this 指向声明时所在作用域下 this 的值（静态this）
4) 箭头函数不能作为构造函数实例化
5) 不能使用 arguments

// 箭头函数适合与 this 无关的回调. 定时器, 数组的方法回调
// 箭头函数不适合与 this 有关的回调.  事件回调, 对象的方法

let fun = () => {
    console.log("hello");
}
fun();  // 输出 hello

const n = 1;
let fun = n => {
    console.log(n);
}
fun(n);  //输出1 如果形参只有一个 那可以省略()，但是如果没有形参那必须要有();

const fun2 = n => n*n;
console.log(fun2(3));  //输出 9 ；如果箭头函数体只有一句代码则可以省略花括号，此时必须省略return 关键字，语句的执行结果就是函数的返回值

const Person = (name,age) => {
    this.name = name;
    this.age = age;
}
const person = new Person("ZK",12);  //报错 Person is not a constructor 箭头函数不能作为构造函数使用；

const fun = () =>{
    console.log(arguments);
}
function fun2() {
    console.log(arguments);
}
fun2("LK");  //输出 Arguments ['LK', callee: ƒ, Symbol(Symbol.iterator): ƒ]
fun("LK");  //报错 arguments is not defined 箭头函数中不能使用 arguments

const name = "LK";
function fun(){
    console.log(this.name);
}
const fun2 = () => {
    console.log(this.name);
}
const obj = {
    name:"ZK",
}

fun();  //输出 LK
fun2(); //输出 LK
fun.call(obj);  //输出 ZK
fun2.call(obj);  //输出 LK  无法通过call改变箭头函数的静态this

箭头函数看上去是匿名函数的一种简写，但实际上，箭头函数和匿名函数有个明显的区别：箭头函数内部的this是词法作用域，由上下文确定。（词法作用域就是定义在词法阶段的作用域。换句话说，词法作用域是由你在写代码时将变量和块作用域写在哪里来决定的，因此当词法分析器处理代码时会保持作用域不变 。）


const Person = {
    firstName : "hello",
    lastName : "world",
    getFullName : function(){
        console.log(this);
        let first = this.firstName;
        const fun = function(){
            console.log(this);
            return this.firstName + this.lastName;
        }
        return fun();
    }
}

let str = Person.getFullName();
//输出 Person对象 window全局对象
console.log(str);   //输出 NaN

const Person = {
    firstName : "hello",
    lastName : "world",
    getFullName : function(){
        console.log(this);
        let first = this.firstName;
        const fun = () => {
            console.log(this);
            return this.firstName + this.lastName;
        }
        return fun();
    }
}

let str = Person.getFullName();
//输出 Person对象 Person对象
console.log(str);   //输出 helloworld
