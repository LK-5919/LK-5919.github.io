CSS预编译处理器 — Less

在进行BootStrap的栅格系统源码分析之前，先对Less进行简单复习。

注释

使用less编写的样式文件并不能直接被浏览器使用，我们需要将此文件提前编译成css文件，其中在使用注释时以下两种注释的方式

方式1：块级注释 -- 编译后会保留在CSS文件中
/*
块级注释
会出现在CSS文件中
*/

方式2：单行注释 -- 编译后不会保留在CSS文件中
//这是一行注释，编译后不可见！

变量

在less中，使用 @ 符号来声明一个变量，如下

//声明一个宽度值
@w: 200px;

//使用变量
.box{
    width: @w;
}

//编译后CSS文件如下：
.box{
    width: 200px;
}

当变量的值只是作为要使用的值的一部分使用时，需要使用 @{变量名} 的形式使用，如下：

//定义一个类名变量
@className: box;

//使用类名变量,其中类名变量作为此时类选择器的一部分值是需要使用{}
.@{className}{
    width: 100px;
}

//定义一个图片基础路径URL变量
@baseImgURL: asset/img/;

//使用变量
body{
    background: url('@{baseImgURL/background.jpg}');
}

在less中，定义的变量存在块级作用域，且存在延迟加载的现象，如下：

//less文件如下
@value: 0;
.outter{
    @value: 1;
    .inner{
        @value: 2;
        inner: @value;
        @value: 3;
    }
    outter: @value;
}

//编译后的css文件如下
.outter{
    outter: 1;
}
.outter .inner{
    inner: 3;
}

/*
*从上面的代码可以知道，当less文件编译时，读取到变量时并不会立刻加载当前变量的值，而是加载完当前块级作用域下的所有变量声明，使用此变量的最后一次赋值，如果当前作用域下未寻找到变量，则沿着作用域链往外寻找。
*/

在，less中，可是引用某一属性值作为变量值，使用 $ 引用属性值，如下

//less文件如下
.box{
    color: black;
    background-color: $color;
}

//编译后的css文件如下：
.box{
    color: black;
    background-color: black;
}

嵌套

在less中，可使用嵌套来使编程更加清晰，可以在选择器中再使用选择器来表明元素间的父子关系，如下

//less文件如下
.outter{
    color: black;
    .innerOne{
        color: red;
    }
    .innerTwo{
        color: blue;
    }
}

//编译后的css文件如下
.outter{
    color: black;
}
.outter .innerOne{
    color: red;
}
.outter .innerTwo{
    color: blue;
}

再less中，使用 & 符号表示父选择器，常用来给当前的选择器添加伪类，如下

//less文件如下
.box{
    color: red;
    
    &:hover{
        color: black;
    }
}

//编译后的css文件如下
.box{
    color: red;
}
.box:hover{
    color: black;
}

& 符还可以用于生成重复的父类名，如下

//less文件如下
.button {
  &-ok {
    background-image: url('ok.png');
  }
  &-cancel {
    background-image: url('cancel.png');
  }

  &-custom {
    background-image: url('custom.png');
  }
}

//编译后的CSS文件如下
.button-ok {
  background-image: url('ok.png');
}
.button-cancel {
  background-image: url('cancel.png');
}
.button-custom {
  background-image: url('custom.png');
}

混合 — Mixin

普通混合 — 将公共样式抽离出单独形成一个类，不使用 () 定义混合时，编译后此公共类也将存在css文件中，如下

//less文件如下
.bordered {
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}

#menu a {
  color: #111;
  .bordered();
}

//编译后的css文件如下
.bordered {
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}
#menu a {
  color: #111;
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}

带参数的混合 – 使用 () 定义公共样式类，编译后的公共样式类不存在css文件中，如下

//less文件如下
.border(@width, @style, @color) {
  border: @width @style @color;
}
div {
  .border(1px, solid, #ccc);
}

//编译后的css文件如下
div {
  border: 1px solid #ccc;
}

注意：使用带参数的混合时，需要按序传入参数

带默认值的混合，使用带默认值的混合时，也需要按序传递参数，如下

//less文件如下
.border(@width, @style, @color: #ccc) {
  border: @width @style @color;
}
div {
  .border(1px, solid);
}

//编译后的css文件如下
div {
  border: 1px solid #ccc;
}

命名参数 — 在使用混合时，可是给实参命名，从而达到不需要按序传入参数的目的，如下

//less文件如下
.border(@width, @style, @color: #ccc) {
  border: @width @style @color;
}
div {
  .border(@style: solid, @color: red, @width: 100px);
}

//编译后的css文件如下
div{
    border: 100px solid red;
}

@arguments变量 — 在使用混合时，可通过@arguments变量获取传入的所有参数（类似js的arguments参数），如下

//less文件如下
.box-shadow(@x: 0, @y: 0, @blur: 1px, @color: #000) {
  -webkit-box-shadow: @arguments;
  -moz-box-shadow: @arguments;
  box-shadow: @arguments;
}
.big-block {
  .box-shadow(2px, 5px);
}

//编译后的css文件如下
.big-block {
  -webkit-box-shadow: 2px 5px 1px #000;
  -moz-box-shadow: 2px 5px 1px #000;
  box-shadow: 2px 5px 1px #000;
}

注意@arguments参数也是按传入的顺序获取

匹配模式 — 在参数中定义一个标志位用来匹配混合，如下

//less文件如下，其中darken和lighten是less的内置混合
.mixin(dark; @color) {
  color: darken(@color, 15%);
}
.mixin(light; @color) {
  color: lighten(@color, 15%);
}

@color-new: dark;

.line {
  .mixin(@color-new; #FF0000);
}

//编译后的css文件如下
.line {
  color: #b30000;
} 

在上面的代码中，dark和light就是标志参数

运算

在less中，可是使用计算表达式表达数值，但经编译后，css文件中将显示具体的数值，如下

@conversion-1: 5cm + 10mm; // 6cm
@conversion-2: 2 - 3cm - 5mm; // -1.5cm
@conversion-3: 3.1 * 2cm; // 6.2cm
@conversion-4: 4px / 2; // 4px / 2

// conversion is impossible
@incompatible-units: 1cm - 1px; // 0.97354167cm

// example with variables
@base: 5%;
@filler: @base * 2; // 10%
@other: @base + @filler; // 15%

@color: #224488 / 2; // #112244
background-color: #112244 + #111; // #223355

继承

继承可以让多个选择器应用同一组样式属性，经编译后为并集选择器，继承的性能比混合高，但是灵活性比混合低，如下

//less文件如下
nav ul {
  &:extend(.inline);
  background: blue;
}
.inline {
  color: red;
}

//编译后的css文件如下
nav ul {
  background: blue;
}
.inline,
nav ul {
  color: red;
}

使用继承中的 all 关键字 ，可以继承指定的所有属性，包括其状态属性，如hover状态等；

  • 可理解为把 all 前的选择器出现的地方全都替换为 extend 前的选择器
  • 即把 .test 替换为 .replacement 生成一个新的选择器应用样式，且不破坏原有的选择器
//less文件如下
.a.b.test,
.test.c {
  color: orange;
}
.test {
  &:hover {
    color: green;
  }
}

.replacement:extend(.test all) {
}

//编译后的css文件如下
.a.b.test,
.test.c,
.a.b.replacement,
.replacement.c {
  color: orange;
}
.test:hover,
.replacement:hover {
  color: green;
}

避免编译

通过加~和引号可以避免 Less 编译，直接把内容输出到 CSS 中，如下

//less文件如下
.textarea1{
    margin: 10 * 10px;
    padding: ~"10 * 10px";
}

//编译后css文件如下
.textarea1 {
  margin: 100px;
  padding: 10 * 10px;
}

BootStrap栅格系统分析

容器

1.流体容器 — container-fluid

流体容器的宽度相当于100%，即容器的宽度与页面的宽度保持一致

2.固定容器 — container

固定容器是有阈值的，即容器在某一阶段的宽度是固定的，不会随页面的大小变化而变化。具体的阈值分类如下

（1）页面宽度小于768px：容器宽度为auto — 移动端 — xs

（2）页面宽度大于等于768px，小于992px时，容器的宽度固定为750px(720px+两边各一半的槽宽30px) — 平板 — sm

（3）页面的宽度大于等于992px，小于1200px时，容器的宽度为970px（940px+两边各一半的槽宽30px）– 中屏pc– md

（4）当页面的宽度大于等于1200px时，容器的宽度为1170px（1140px+两边各一半的槽宽30px）— 大屏PC — lg

栅格系统

在bootstrap中，栅格系统的样式由grid.less文件定义，查看grid.less文件，如下：

//固定容器
.container {
    //栅格容器的公共基础样式
  .container-fixed();
  @media (min-width: @screen-sm-min) {
    width: @container-sm;
  }
  @media (min-width: @screen-md-min) {
    width: @container-md;
  }
  @media (min-width: @screen-lg-min) {
    width: @container-lg;
  }
}
//流体容器
.container-fluid {
  .container-fixed();
}
// 行
.row {
  .make-row();
}
// 列
.make-grid-columns();
// 移动优先
.make-grid(xs);
@media (min-width: @screen-sm-min) {
  .make-grid(sm);
}
@media (min-width: @screen-md-min) {
  .make-grid(md);
}
@media (min-width: @screen-lg-min) {
  .make-grid(lg);
}

通过variables.less文件查看bootstrap的变量定义，可知

@grid-gutter-width: 30px; --- 槽宽

//平板的栅格宽度
@container-tablet: (720px + @grid-gutter-width);
@container-sm: @container-tablet;

//大屏的栅格宽度
@container-desktop: (940px + @grid-gutter-width);
@container-md: @container-desktop;

//超大屏的栅格宽度
@container-large-desktop: (1140px + @grid-gutter-width);
@container-lg: @container-large-desktop;

//Phone
@screen-xs: 480px;
//tablet
@screen-sm-min = @screen-sm: 768px;
//desktop
@screen-md-min = @screen-md: 992px;
//wide desktop
@screen-lg-min = @screen-lg: 1200px;

@screen-xs-max: (@screen-sm-min - 1);
@screen-sm-max: (@screen-md-min - 1);
@screen-md-max: (@screen-lg-min - 1);

//栅格默认列数
@grid-columns: 12;

通过上面的基本变量可知，grid.less文件先定义了固定容器的样式，其中，在里面又调用了固定容器与流体容器均有的公共样式混合类container-fixed，其具体代码如下：

// 固定和流体容器的公共样式
//@grid-gutter-widt：槽宽
.container-fixed(@gutter: @grid-gutter-width) {
  margin-right: auto;
  margin-left: auto;
  padding-left:  floor((@gutter / 2));  //向下取整
  padding-right: ceil((@gutter / 2));  //向上取整
  &:extend(.clearfix all);
}

/*
	可知调用容器的的公共样式类时，不传入槽宽值，则使用默认槽宽30px
	公共样式类给容器开启了左右margin为auto
	容器的左右内边距paddind为槽宽的一半15px
	给容器继承清除浮动样式，清除浮动具体代码如下
*/

.clearfix() {
  &:before,
  &:after {
    content: " "; // 1
    display: table; // 2
  }
  &:after {
    clear: both;
  }
}

在设置完容器的基本样式后，对固定容器的进行媒体查询，如下：

//固定容器
.container {
    //设置公共样式
  .container-fixed();
    //媒体查询，当屏幕宽度大于等于768px时
  @media (min-width: @screen-sm-min) {
      //设置容器的宽度为750px
    width: @container-sm;
  }
    //媒体查询，当屏幕宽度大于等于992px时
  @media (min-width: @screen-md-min) {
      //设置容器的宽度为970px
    width: @container-md;
  }
    //媒体查询，当屏幕宽度大于等于1200px时
  @media (min-width: @screen-lg-min) {
      //设置容器的宽度为1170px
    width: @container-lg;
  }
}

在固定容器的样式设置完毕以后，开始设置流体容器的样式，流体容器就直接调用了容器的基本公共样式container-fixed类，没有进行媒体查询设置流体容器的宽度，此时流体容器的宽度为auto；

补充：宽度100%和宽度auto的区别：元素的宽度100%是指，元素的宽度此时和页面大小一致，此时给元素设置内边距时会影响元素的可见盒子大小；而宽度为auto时，设置元素的内边距时，元素的内容区将缩小，而元素的可见盒子大小整体不变。

行

在设置完固定容器和流体容器的样式后，开始设置栅格中行的样式，如下：

// 行
.row {
    //不传入参数，使用默认槽宽30px
  .make-row();
}

/*
	可见在行的样式定义中使用了混合 make-row类，具体代码如下
*/

// 行
.make-row(@gutter: @grid-gutter-width) {
  margin-left:  ceil((@gutter / -2));
  margin-right: floor((@gutter / -2));
  &:extend(.clearfix all);
}

/*
	从上面的代码可以知道，给栅格中的行设置了左右各-15px的外边距
	以及继承清除浮动的样式
*/

列

在设置完行的样式后，开始设置行中单元格的样式，即列的样式，如下：

// 列样式设置

//列样式设置第一步
//设置列的基本样式，全平台通用的公共列样式
.make-grid-columns();

//列样式设置第二步
// 移动优先 --- xs --- 设置移动端的列样式
.make-grid(xs);

//开启媒体查询
@media (min-width: @screen-sm-min) {
  //设置平板端的列样式
  .make-grid(sm);
}

@media (min-width: @screen-md-min) {
    //设置小屏PC端的列样式
  .make-grid(md);
}
@media (min-width: @screen-lg-min) {
    //设置大屏PC端的列样式
  .make-grid(lg);
}

//首先是调用make-grid-columns混合，具体代码如下：
// 列第一步
.make-grid-columns() {
  .col(@index) { 
    @item: ~".col-xs-@{index}, .col-sm-@{index}, .col-md-@{index}, .col-lg-@{index}";
    .col((@index + 1), @item);
  }
  .col(@index, @list) when (@index =< @grid-columns) { 
    @item: ~".col-xs-@{index}, .col-sm-@{index}, .col-md-@{index}, .col-lg-@{index}";
      //递归调用自己
    .col((@index + 1), ~"@{list}, @{item}");
  }
  .col(@index, @list) when (@index > @grid-columns) { 
    @{list} {
      position: relative;
      min-height: 1px;
      padding-left:  ceil((@grid-gutter-width / 2));
      padding-right: floor((@grid-gutter-width / 2));
    }
  }
  .col(1); 
}

/*
	在make-grid-columns类中定义了三个col类（参数类型不同），并在最后调用了col类
	在这里对调用col(1)混合背后所执行的步骤进行详细说明：
	（1）调用混合 col(1), 传入参数1，此时将执行此块级作用域下的第一个混合.col(@index)
	（2）执行混合 col(1)，此时的@item变量的值为 ".col-xs-1, .col-sm-1, .col-md-1, .col-lg-1"
		接着继续调用混合 col(@index, @list) when (@index =< @grid-columns)
	（3）执行混合 .col(2, .col-xs-1, .col-sm-1, .col-md-1, .col-lg-1)，并且开始循环递归
		当@index  <= 12 时，循环调用此混合，此时的@index = 2
		每循环一次，@index自增1，且将传入的@item与新@item直接使用 , 拼接
		...
		当循环至@index = 13时，此时结束循环，调用混合 col(@index, @list) when (@index > @grid-columns)
	（4）当执行完上面的循环后，将@item变量中的字符串作为并集选择器使用，效果如下
		.col-xs-1, .col-sm-1, .col-md-1, .col-lg-1,
         .col-xs-2, .col-sm-2, .col-md-2, .col-lg-2,
         ...
         .col-xs-12, .col-sm-12, .col-md-12, .col-lg-12{
			//开启相对定位
			 position: relative;
			//设置最小高度
              min-height: 1px;
			//设置左内边距为15px
              padding-left:  ceil((@grid-gutter-width / 2));
			//设置右内边距为15px
              padding-right: floor((@grid-gutter-width / 2));
         }
*/


//当列设置的第一步执行完毕以后，开始进行列第二步的样式设置
//设置移动端的列样式
.make-grid(@class) {
    //此时@class = xs
    //2.1步,设置各种列类型开启左浮动
  .float-grid-columns(@class);
    //2.2步，设置各种列类型所占容器的宽度
  .loop-grid-columns(@grid-columns, @class, width);
    //2.3步
    //(列排序)，记忆左推右拉
    //2.3.1步，设置各种列类型相对自身定位的移动的right值
  .loop-grid-columns(@grid-columns, @class, pull);
    //2.3.2步，设置各种列类型相对自身定位移动的left值
  .loop-grid-columns(@grid-columns, @class, push);
    //2.4步
    //(列偏移),设置各种列类型的左外边距值,使各种列向右偏移
  .loop-grid-columns(@grid-columns, @class, offset);
}

/*
	2.1步 --- 在设置移动端的列样式时，首先调用了混合.float-grid-columns(xs)，代码如下
*/
.float-grid-columns(@class) {
  .col(@index) { 
    @item: ~".col-@{class}-@{index}";
    .col((@index + 1), @item);
  }
  .col(@index, @list) when (@index =< @grid-columns) { 
    @item: ~".col-@{class}-@{index}";
    .col((@index + 1), ~"@{list}, @{item}");
  }
  .col(@index, @list) when (@index > @grid-columns) { 
    @{list} {
      float: left;
    }
  }
  .col(1); 
}
/*
	2.1步代码详解：
	（1）首先调用混合 col(1)，此时@item = .col-xs-1 ，接着调用混合 col(2,.col-xs-1)
	（2）开始循环，当循环至@index = 13 时，结束循环
	（3）将item中的值作为并集选择器，并为这些选择器开启左浮动，具体效果如下：
		.col-xs-1,.col-xs-2,
    	.col-xs-3,.col-xs-4,
    	...,
    	col-xs-12{
      		float: left;
     	}
*/

//进行2.2步，调用混合 loop-grid-columns(@grid-columns, @class, width)，具体代码如下：
.loop-grid-columns(@index, @class, @type) when (@index >= 0) {
    //首次调用时 @index = 12， @class = xs , @type = width
    //2.2.1步，设置列所占容器的宽度
  .calc-grid-column(@index, @class, @type);
    //2.2.2步，循环递归调用自己
  .loop-grid-columns((@index - 1), @class, @type);
}

//2.2.1步，代码如下
.calc-grid-column(@index, @class, @type) when (@type = width) and (@index > 0) {
  .col-@{class}-@{index} {
    width: percentage((@index / @grid-columns));
  }
}
/*
	2.2步代码执行步骤详解：
	（1）刚开始执行时 @index = 12， @class = xs , @type = width ，开始循环，知道@index = 0 时，结束循环
	（2）每次循环，都给当前选择器设置一个列宽度
	（3）最后2.2步循环结束后的效果为：
     .col-xs-12{ width: 12/12;}
	.col-xs-11{ width: 11/12;}
	...
	.col-xs-1{ width: 1/12;}
*/

//2.3步的入口代码同2.2步入口代码一致，不同的是@type参数值不同，此时内部执行的混合也不同，代码如下
.calc-grid-column(@index, @class, @type) when (@type = push) and (@index > 0) {
  .col-@{class}-push-@{index} {
    left: percentage((@index / @grid-columns));
  }
}
.calc-grid-column(@index, @class, @type) when (@type = push) and (@index = 0) {
  .col-@{class}-push-0 {
    left: auto;
  }
}
.calc-grid-column(@index, @class, @type) when (@type = pull) and (@index > 0) {
  .col-@{class}-pull-@{index} {
    right: percentage((@index / @grid-columns));
  }
}
.calc-grid-column(@index, @class, @type) when (@type = pull) and (@index = 0) {
  .col-@{class}-pull-0 {
    right: auto;
  }
}

/*
	以上的代码执行完毕后，效果如下：
		列左排序
		.col-xs-push-12{ left: 12/12;}
		.col-xs-push-11{ left: 11/12;}
		...
		.col-xs-push-1{ left: 1/12;}
		.col-xs-push-0{ left: auto;}
		列右排序
		.col-xs-pull-12{ right: 12/12;}
		.col-xs-pull-11{ right: 11/12;}
		...
		.col-xs-pull-1{ right: 1/12;}
		.col-xs-push-0{ right: auto;}
*/

//2.4步入口代码同2.2步，但是传入的@type值 为 offset，此时内部执行的混合不同，代码如下：
.calc-grid-column(@index, @class, @type) when (@type = offset) {
  .col-@{class}-offset-@{index} {
    margin-left: percentage((@index / @grid-columns));
  }

bootstrap盒子模型设计的精妙之处

从上面的代码可以知道，栅格的固定容器外部两边各有15px的padding，行的两边各有-15px的margin，列的两边各有15px的padding作为槽宽；

从内往外分析，列两边具有15px的padding槽宽是为了列与列之间的内容不会紧紧相连在一起，如下

列与列之间有了15px的padding槽宽，但如果在列之间再放入行，行之间再放入列，效果如下图：

在第一列中再放入一行，且保持列与列之间的15px槽宽，此时，就会造成列中嵌套的行中的列与最外部的列之间间隔为45px，造成间隔过大，所以此时需要给行设置两边各-15px的margin，如下图：

如上图，给行的两边加上-15px的margin后，此时最外部的行两侧也就多了-15px的外边距，为了不影响固定容器的整体可见盒子大小，需要给最外边的固定容器两边加上15px的内边距，从而抵消行的负外边距。
