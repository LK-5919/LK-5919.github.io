模块化

是什么

将一个复杂的程序文件按照业务逻辑划分为多个单个文件，最终再组合在一起，拆分的文件就是模块，模块内部的数据是私有的，只是向外部暴露一些方法与外部其他模块通信；

模块化的优点

  • 避免命名冲突
  • 更好的分离，实现按需引入加载
  • 提高代码复用性和提高解耦性

模块化的解决方案

  • CommonJS（可运行再服务端node和浏览器端）
  • ES6（只能运行在浏览器端）

CommonJS

  1. CommonJS的模块化代码可以直接运行在服务端运行；
  2. CommonJS的模块化代码需要经过browserify编译后才能运行在浏览器端；

基本语法

//向外暴露模块 --- export

方法1： module.exports = value

方法2： exports.xxx = value

//引入模块 --- require

引入自定义模块 ： require('./xxx'); //xxx为自定义模块文件所在的位置
引入第三方模块 ： require('xxx'); // xxx为第三方模块的名字

在CommonJS中，module.exports 和 exports 是存放在栈中的引用，引用的内容是存放在堆中的数据，也就是说每一个模块化文件都有自己的module.exports 和 exports，初始情况下它们引用的是堆内存中的一个空对象；如下图：

暴露和引入自定义模块

使用方法一向外暴露一个方法

const name = "LK";

//使用方法1向外暴露一个方法,在这里将会在队中新开辟一片内存空间，并将module.exports的引用值指向此新对象
module.exports = {
    getName(){
        console.log(name);
    }
}

运行结果如下

可知以上的方法module.exports = {}将会是重新划分一个新对象，但是如果使用 module.exports.xxx = value，将会在原来初始的空对象中添加属性，此时module.exports 和exports指向的仍然是同一个对象，如下：

//在原有的暴露空对象中追加属性
module.exports.getName = function (){
    console.log(name);
}

console.log(module.exports);
console.log(exports);

运行结果如下

使用方法2进行方法暴露

const msg = "hello";

//使用方法2向外暴露方法,此时将会在堆中重新开辟空间存放暴露的方法
exports = {
    getMsg(){
        console.log(msg);
    }
}

console.log(module.exports);
console.log(exports);

运行结构如下：

但如果使用exporst.xxx = value将会在原来初始的空对象上添加属性，如下

const msg = "hello";
//在初始的空对象上添加属性
exports.getMsg = function(){
    console.log(msg);
}

console.log(module.exports);
console.log(exports);

module1.getName(); //输出 LK

CommonJS运行在浏览器端

commonJS中的require命令在浏览器中是不可识别的，直接使用将报错，如下，新建index.html文件，在里面引入app.js文件，如下图：

此时，需要使用browserify将使用commonjs规范的文件编译成浏览器可以识别的文件，步骤如下

  • npm全局安装browserify 命令： cnpm i browserify -g
  • 安装完成后，进入需要编译文件 的所在目录
  • 控制台输入 browserify app.js -o build.js

此时，在index.html文件中引入build.js文件，网页即可正常运行（其中 -o 是output的意思，一般会把编译后的文件命名为build.js）

ES6模块化

ES6提供的模块化语法只能在浏览器端运行，但是浏览器并不认识ES6语法，需要先使用babel将所有使用了ES5语法的js文件转换成CJS（CommonJS）语法的文件，之后再使用browserify将转换后的CJS入口文件app.js转换成浏览器可识别的build.js文件，Babel使用方法如下：

  • 先全局安装babel-cli（command line interface）npm i babel-cli -g;
  • 局部安装babel-preset-es2015 cnpm install babel-preset-es2015;
  • 再使用ES6模块化的文件根目录下新建.babelrc文件，其中rc代表 run control（运行时控制），写入如下代码：
{
	"presets":["es2015"]
}

经过以上步骤，就可开始使用ES6模块化语法，新建src文件夹，在里面书写页面文件和js文件；

ES6模块化语法 — 分别暴露

新建module1.js文件，使用ES6模块化语法分别暴露模块，代码如下：

//此文件使用ES6分别暴露语法暴露模块

export const name = "ZK";

export const msg = "hello";

export function getName(){
    console.log(name);
}

export function getMsg(){
    console.log(msg);
}

新建app.js文件，使用ES6模块化语法引入模块，代码如下

//此文件为使用ES6模块化语法的合并文件

import { name,msg,getName,getMsg } from "./module1";

console.log(name);

console.log(msg);

getName();

getMsg();

至此，页面是不能正确使用app.js文件的，在index.html文件中引入app.js脚本，报错如下：

可知，浏览器是不能直接识别ES6的模块化语法的，现在开始使用babel将src文件夹下所有使用了ES6模块化语法的js文件转换成CJS语法的文件，命令如下：babel ./src -d ./build ,将转换后的所有js放在同目录下的build文件夹，此时build文件夹中的文件任不能被浏览器使用，需要使用browserify将入口文件app.js进行转换，输入如下命令：browserify ./build/app.js -o ./build/build.js; 之后再index.html文件中引入转换后的build.js文件，运行解果如下：

注意：分别暴露import接受时使用{ }，并不能直接把它理解成为解构赋值（CJS语法在引入时可以使用解构辅助的语法，因为其等号右边是对象，但是ES6语法不是）；

延申一下分别暴露的另外两种形式：

  • 当多个模块分别暴露的内容命名冲突时，采用如下重命名方案：
import {data,msg} from './module1';
import {data as data2, msg as msg2} from './module2';

//将module2中与module1中重名的内容使用 as 关键字重命名
  • 直接将分别暴露的内容使用对象形式接受，并重命名：
import * as module1 from './module1';

console.log(module1);

ES6模块化语法 — 统一暴露

src文件中新建module2.js文件，在文件中使用统一暴露语法暴露模块，代码如下：

//此文件使用 统一暴露语法暴露模块

const name = "LK";
const person = {
    age: 20,
    sex: "男"
}
function getPerson(){
    console.log(person);
}

//统一暴露，注意统一暴露时使用的{} ,并不意味着暴露就是一个对象，这只是ES6语法而已，
//在引入统一暴露的模块时，引入的方法和分别暴露引入方法一致
//统一暴露于分别暴露的区别在于，可以在最后批量暴露， 且在暴露的时候可以进行重命名

//统一暴露的简写形式
export {name,person,getPerson};

//统一暴露的完整写法,在暴露的时候可进行重命名
// export {name as a, person as p ,getPerson as funP}

在app.js文件中引入module2模块，并重新转换：

//引入统一暴露的模块,方法和引入分别暴露的三种方法一致
import {name,person,getPerson} from './module2';

console.log(name);
console.log(person);
getPerson();

补充：引入统一暴露的模块方法和引入分别暴露模块的方法一致，不再赘述；

ES6模块化语法 — 默认暴露

在src文件件中新建module3.js，在此文件中使用默认暴露语法暴露模块，代码如下：

//此文件使用默认暴露语法暴露模块

//默认暴露 使用 export default 关键字,默认暴露在一个文件中只能使用一次

//默认暴露 export default 关键字后需要接表达式

/**
 * 
 * 错误形式
 * export default const a = 1;
 * 
 * 以上的暴露形式是错误的,相当于在接受时 xxx = const a = 1;
 * 
 * 应采取如下形式:
 * export default 1;
 *  以上形式,相当于在接受时 xxx  = 1;
 * 
 * 或则在最后再进行默认暴露,如下
 * 
 * const a = 1;
 * export default a;
 */


//使用默认暴露向外暴露数据

export default {
    name: "LK",
    age: 22
}

再app.js文件中引入默认暴露的模块，代码如下：

//引入默认暴露的模块,引入默认暴露的模块时,直接定义一个变量接收
import module3 from './module3';
console.log(module3);

补充

在CJS中，若在一个模块中同时使用module.exports 和 exports ，将会优先使用前者，但是ES6语法中，可以同时使用多种暴露方法，在src文件夹中新建module4.js文件，同时使用多种暴露方法，如下：

//在此文件中同时使用ES6中的多种模块化语法

//分别暴露
export const name = "LK";

//统一暴露
function sayHello(){
    console.log("hello");
}

export {sayHello};

//默认暴露
export default {
    id: "001",
    work: "WEB"
}

在app.js文件中使用如下方法引入模块：

//引入同时使用多种暴露方法的模块
import module4,{name,sayHello} from './module4';

console.log(module4);
console.log(name);
sayHello();
