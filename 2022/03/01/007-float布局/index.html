007-float布局

2022-03-01 HTML-CSS学习 1.9k 字
WEB前端

1.浮动布局（Float）

浮动： 使用 float 属性设置 默认值为none； left 元素向左移动； right 元素向右移动；
	PS：一旦元素开启了float属性 此元素将脱离文档流 水平布局的等式便不再强制成立
		不占用文档流的元素浮动后 还在文档流的元素将自动补上

 .demo{
            width: 200px;
            height: 200px;
            background-color: #bfa;
            /* 此时没开启float时 块级元素独占一行 盒子的可见框大小为 200px * 200px 浏览器将自动调整margin-right */
            float: left;
            /* 开启float 后 margin-right == 0 元素脱离文档流*/
        }
        .demo1{
            width: 205px;
            height: 205px;
            background-color: green;
            /* 此时demo1自动补上文档流中的demo离开的位置 此时demo1完全被浮动的demo遮挡 不可见 */
        }
<div class="demo"></div>
<div class="demo1"></div>

浮动的特点： 按元素的浮动顺序排列 排在后面的浮动元素所在高度不会超过其前面浮动元素的所在高度
                                排在前面的浮动元素越接近父元素的边界 但无法超出父元素

2.浮动布局与文字

浮动布局最早用来解决文字环绕问题 浮动元素不会遮住文字 文字会自动环绕在浮动元素周围
.demo{
            width: 200px;
            height: 200px;
            background-color: #bfa;
            float: left;
            /* 开启float后 div上浮 但不会遮住span中的文本 */
        }
.content{
           
            float: left;
            /* 可知行内元素开启float 脱离文档流后 特点变得和块元素一样 */
        }

 <span class="content">Lorem ipsum dolor sit amet consectetur adipisicing elit. Molestiae enim officia dolore non dolor eligendi maxime et quis, accusamus laborum pariatur praesentium, saepe dolores ipsa est libero delectus itaque ullam!</span>
 <div class="demo"></div>

.test1{
            background-color: #bfa;
            /* 未开启float时 div默认独占一行 不主动设置宽高的时候 div宽度默认100% 高度由内容撑开 */
            float: left;
            /* 开启float 后 块级元素div 脱离文档流 此时元素的宽高默认由内容撑开 */
        }
 <div class="test1">
        我是一个类名为 test1 的div
        <div class="test2">我是在test1类里的div</div>
  </div>


脱离文档流的特点：
                    块元素：
                        1、块元素不在独占页面的一行
                        2、脱离文档流以后，块元素的宽度和高度默认都被内容撑开

                    行内元素：
                        行内元素脱离文档流以后会变成块元素，特点和块元素一样

                    脱离文档流以后，不需要再区分块和行内了

PPS ： float元素 默认宽高均由内容撑开 但也可以手动设置宽高 此时不再区分块元素和内联元素

3.高度塌陷问题

在普通布局中 外层的块级元素高度默认由内部的块级元素高度撑开 但若父元素内部的子元素开启浮动后 父元素高度将丢失

        .box1{
            /* 1.不设置宽高 默认其由子元素撑开 */
            /* 2.设置边框 */
            border: 5px red solid;
        }
        .box2{
            width: 200px;
            height: 200px;
            background-color: #bfa;
            /* 3.此时子元素将父元素撑开 */
            float: left;
            /* 4.开启子元素浮动后 子元素脱离文档流 将无法撑开父元素高度 */
        }
	 <div class="box1">
        <div class="box2"></div>
    </div>

4.BFC（块级格式化环境）解决高度塌陷问题

 BFC(Block Formatting Context) 块级格式化环境
                    - BFC是一个CSS中的一个隐含的属性，可以为一个元素开启BFC
                        开启BFC该元素会变成一个独立的布局区域
                    - 元素开启BFC后的特点：
                        1.开启BFC的元素不会被浮动元素所覆盖
                        2.开启BFC的元素子元素和父元素外边距不会重叠
                        3.开启BFC的元素可以包含浮动的子元素

                    - 可以通过一些特殊方式来开启元素的BFC：
                        1、设置元素的浮动（不推荐）
                        2、将元素设置为行内块元素（不推荐）
                        3、将元素的overflow设置为一个非visible的值
                            - 常用的方式 为元素设置 overflow:hidden 开启其BFC 以使其可以包含浮动元素
 .box1{
            /* 1.不设置宽高 默认其由子元素撑开 */
            /* 2.设置边框 */
            border: 5px red solid;
            overflow: hidden;
            /* 5.父元素开启 BFC 使其可以包含浮动元素 此时父元素不会被浮动元素遮挡*/
            
        }
        .box2{
            width: 200px;
            height: 200px;
            background-color: #bfa;
            /* 3.此时子元素将父元素撑开 */
            float: left;
            /* 4.开启子元素浮动后 子元素脱离文档流 将无法撑开父元素高度 浮动的子元素遮挡住了父元素*/
        }
 <div class="box1">
        <div class="box2"></div>
 </div>


BFC   解决外边距重叠问题
 .demo1{
            width: 400px;
            height: 400px;
            background-color: #bfa;
            overflow: hidden;
            /* 3.父元素开启BFC 父元素的外边距与子元素的外边距不重叠 实现子元素在父元素中下移 而父元素不下移 */
        }
        .demo2{
            width: 200px;
            height: 200px;
            background-color: skyblue;
            /* 1.此时子元素与父元素的外边距重叠 */
            margin-top: 200px;
            /* 2.此时父元素 与 子元素 一起下移*/
        }

 <div class="demo1">
        <div class="demo2"></div>
    </div>

5.clear

在普通的块级元素的垂直布局中 若文档流中有元素由于浮动脱离文档流后 该元素之后的文档流元素位置将发生移动 
	若不希望元素受到浮动的影响 可使用clear 属性清除浮动元素产生的影响
clear
                    - 作用：清除浮动元素对当前元素所产生的影响
                    - 可选值：
                        left 清除左侧浮动元素对当前元素的影响
                        right 清除右侧浮动元素对当前元素的影响
                        both 清除两侧中最大影响的那侧

                    原理：
                        设置清除浮动以后，浏览器会自动为元素添加一个上外边距，
                            以使其位置不受其他元素的影响

 		.demo1{
            width: 400px;
            height: 200px;
            background-color: #bfa;
            float: left;
            /* 1.此时demo1开启浮动 文档流中其后的demo2 和 demo3 位置将发生移动 */
        }
        .demo2{
            width: 200px;
            height: 200px;
            background-color: skyblue;
            clear: left;
            /* 2.demo2 开启clear ：left 清除其前面元素demo1 左浮带来的影响 */
            float: right;
            /* 3.此时demo2 开启右浮 demo3位置发生改变 */
        }
        .demo3{
            width: 400px;
            height: 100px;
            background-color: green;
            clear: right;
            /* demo3 开启 clear ： right 消除demo2右浮的影响 */
        }
<div class="demo1">1</div>
<div class="demo2">2</div>
<div class="demo3">3</div>

6.clearfix解决高度塌陷

.demo{
            /* 1.demo不设置宽高 默认其的高度由子元素撑开 此时默认width为100% */
            border: 5px red solid;
        }
        .test{
            width: 200px;
            height: 200px;
            background-color: #bfa;
            float: left;
            /* 2.开启子元素浮动 父元素高度塌陷 */
        }
        .demo::after{
            content: '';
            display: block;
            clear: both;
            /* 3.使用伪类选择器 在父元素的最后加入内容
                    加入的内容为：
                        1.''空字符串
                        2.将这个字符串转换成block元素
                        3.因为这个字符串元素在浮动元素test后面 使用clear 清除浮动影响
                        使用clear相当于在这个字符串元素身上加上了 margin-top 高度为test的高度 所以此时父元素相当于被这个字符串元素撑开
            
            这里也不推荐使用block 推荐使用table
            */
        }

<div class="demo">
	 <div class="test"></div>
</div>


最终的clearfix样式：
/* clearfix 这个样式可以同时解决高度塌陷和外边距重叠的问题，当你在遇到这些问题时，直接使用clearfix这个类即可 */
        .clearfix::before,
        .clearfix::after{
            content: '';
            display: table;
            clear: both;
        }
PS：这里将字符串元素转换成table类型 是因为block不能将父子元素的外边距分离开
	而采用table可以实现外间距分离和高度塌陷
