008-position布局

1.position布局

​ 定位（position）是一种布局手段 通过定位可以将元素摆放到任意位置

​ 使用position属性来开启定位：

​ 可选值 1.static 默认值 此时元素为不开启定位

​ 2.relative 开启元素相对定位

​ 3.absolute 开启元素绝对定位

​ 4.fixed 开启元素固定定位

​ 5.sticky 开启元素粘滞定位

      偏移量（offset）：元素开启定位后 可以通过偏移量来设置元素的位置

​ top — 定位元素和定位位置上边的距离 top值越大 定位元素越靠下

​ bottom — 定位元素和定位位置下边的距离 bottom值越大 定位元素越靠上

​ left — 定位元素和定位位置左侧的距离 left值越大 定位元素越靠右

​ right — 定位元素和定位位置右侧的距离 right值越大 定位元素越靠左

2.相对定位

当元素的position属性为 relative 时 开启相对定位 
	- 相对定位的特点：
                            1.元素开启相对定位以后，如果不设置偏移量元素不会发生任何的变化
                            2.相对定位是参照于元素在文档流中的位置进行定位的
                            3.相对定位会提升元素的层级
                            4.相对定位不会使元素脱离文档流
                            5.相对定位不会改变元素的性质块还是块，行内还是行内
.demo1{
            width: 200px;
            height: 200px;
            background-color: #bfa;
            /* 1.初始三个div垂直排列 */
        }
        .demo2{
            width: 200px;
            height: 200px;
            background-color: skyblue;
            position: relative;
            /* 2.demo2开启相对定位 但是还未设置偏移量
                开启相对定位的元素不会脱离文档流 所以demo3的位置不会发生变化
            */
            left: 0px;
            right: 0px;
            /* 3.给元素的相对定位偏移量设置为 left 0 right 0 让其回到参照原点 可知
                相对定位的参照原点为其在文档流时候的位置
            */
            left: 200px;
            top: -200px;
            /* 4.给元素依照参照原点 向右移动200px 向上移动200px 使其与demo1水平相邻排列*/
            left: 70px;
            /* 5.将元素demo2向右水平移动的距离改为70px 可见现在demo2遮盖住了demo1 所以可知相对定位会提升元素的层级
                应为开启相对定位的元素未脱离文档里 不会改变元素的块级元素性质 而且开启相对定位的元素 其文档流后的元素不会因为其移动为改变布局
            */
        }
        .demo3{
            width: 200px;
            height: 200px;
            background-color: green;
            /* demo3全程不移动 虽然demo已经移走 但是demo2还占据文档流中的位置 */
        }
	<div class="demo1">1</div>
    <div class="demo2">2</div>
    <div class="demo3">3</div>

3.绝对定位

当元素的position属性值为 absolute 时 元素开启绝对定位
	- 绝对定位的特点：
                    1.开启绝对定位后，如果不设置偏移量元素的位置不会发生变化
                    2.开启绝对定位后，元素会从文档流中脱离
                    3.绝对定位会改变元素的性质，行内变成块，块的宽高被内容撑开
                    4.绝对定位会使元素提升一个层级
                    5.绝对定位元素是相对于其包含块进行定位的
包含块( containing block )
                        - 正常情况下：
                            包含块就是离当前元素最近的祖先块元素
                            <div> <div></div> </div>
                            <div><span><em>hello</em></span></div>

                        - 绝对定位的包含块:
                            包含块就是离它最近的开启了定位的祖先元素，
                                如果所有的祖先元素都没有开启定位则根元素就是它的包含块

                        - html（根元素、初始包含块）
.demo1{
            width: 200px;
            height: 200px;
            background-color: #bfa;
        }
        .demo2{
            width: 200px;
            height: 200px;
            background-color: skyblue;
            /* 1.初始demo1和demo2垂直相邻排列 */
            position: absolute;
            /* 4.demo2 开启绝对定位 demo2脱离文档流 所以demo3向上移动部位
                但是demo2还未设置偏移量 所以demo2位置不发生变化
                但是明显demo2遮挡住了demo3 可知开启绝对提升了元素的层级
            */
            top: 0;
            left: 0;
            /* 5.让demo2回到定位参照原点 可知此时demo2回到了最左上角 */
            /* 8. 可知demo2此时的包含块是html根标签 所以回到最左上角 */
        }
        .demo3{
            width: 400px;
            height: 400px;
            background-color: greenyellow;
        }
        .demo4{
            width: 300px;
            height: 300px;
            background-color: pink;
            position: relative;
            /* 6.demo4 开启相对定位  未设置偏移量 此时demo4位置不发生变化*/
        }
        .demo5{
            width: 200px;
            height: 200px;
            background-color: gold;
            /* 2.初始demo3里包含demo4，demo4包含demo5 */
            position: absolute;
            /* 3.demo5开启绝对定位 还未设置偏移量 页面布局不发生变化 */
            top: 0;
            left: 0;
            /* 7.让demo5回到定位参照原点 可见demo5回到了demo4文档流所在位置
                可知绝对定位的参照原点是距其最近的开启了定位的包含块
                    若元素的祖先们均未开启定位 则该元素的定位参照原点为html根包含块
            */
        }

<div class="demo1">1</div>
    <div class="demo2">2</div>
    <div class="demo3">3
        <div class="demo4">4
            <div class="demo5">5</div>
        </div>
    </div>

PS：实际开发中 使用绝对定位和相对定位较多

4.固定定位

元素的position属性值为 fixed 时 元素开启固定定位
	特点：固定定位为绝对定位的一种 用法性质基本和绝对定位一致 
			唯一不同的是固定定位的参照原点一直都是浏览器的视口
			固定定位的元素不会随网页的滚动条移动
 .demo1{
            width: 200px;
            height: 200px;
            background-color: #bfa;
        }
        .demo2{
            width: 250px;
            height: 250px;
            background-color: skyblue;
            position: fixed;
            /* 2.demo2开启固定定位 */
            top: 0;
            left: 0;
            /* 3. demo2回到定位参照原点 此时demo2位于页面最左上角 demo2此时脱离文档流 demo3上移*/
        }
        .demo3{
            width: 300px;
            height: 300px;
            background-color: yellowgreen;
            position: relative;
            /* 4.demo3开启相对定位 */
        }
        .demo4{
            width: 200px;
            height: 200px;
            background-color: gold;
            position: fixed;
            /* 1.demo4开启固定定位 此时未设置偏移量时 demo4固定在网页的这个位置不动 */
            top: 0;
            left: 0;
            /* 5.demo4也回到定位参照原点 可知 此时demo4也会在demo2所在位置
                可知固定定位的参照原点是浏览器的网页视口
            */
             left: 400px;
            /* 让demo4从参照原点向右移动400px */
        }
<div class="demo1">1</div>
    <div class="demo2">2</div>
    <div class="demo3">3
        <div class="demo4">4</div>
    </div>

5.粘滞定位

粘滞定位
                    - 当元素的position属性设置为sticky时则开启了元素的粘滞定位
                    - 粘滞定位和相对定位的特点基本一致，
                        不同的是粘滞定位可以在元素到达某个位置时将其固定
粘滞定位只能调整top 和 left 且调整top是不影响元素的原位置 只有在滑动元素要离开这一页面时才生效

  .demo1{
            width: 200px;
            height: 200px;
            background-color: beige;
        }
        .demo2{
            width: 250px;
            height: 250px;
            background-color: skyblue;
            position: sticky;
            /* 1.demo1 开启粘滞定位 特点和相对定位差不多 元素不脱离文档流 */
            left: 0;
            top: 0;
            /* 2.demo2 回到参照原点  此时网页下滑时demo2在浏览器视口位置保持不动*/
            left: 200px;
            /* 3. demo2 向左移动200px 此时网页下滑时 demo2在浏览器视口向左移动200px的位置保持不变 */
            top: 20px;
            /* 4. 未滑动时demo2位置不发生变化 可知设置top未滑动页面时不影响元素原位置 
                在页面下滑时 页面会停留在参照浏览器视口而偏移的位置
            */
        }
        .demo3{
            width: 100px;
            height: 100px;
            background-color: dodgerblue;
        }

<div class="demo1">1</div>
    <div class="demo2">2</div>
    <div class="demo3">3</div>

6.绝对定位元素的布局

开启绝对定位以后 水平方向的布局的等式需要再加入 left right 两个值
 left + margin-left + border-left + padding-left + width + padding-right + border-right + margin-right + right = 包含块的内容区的宽度
此时规则和之前一样只是多添加了两个值：
                            当发生过度约束：
                                如果9个值中没有 auto 则自动调整right值以使等式满足
                                如果有auto，则自动调整auto的值以使等式满足

                        - 可设置auto的值
                            margin width left right

                        - 因为left 和 right的值默认是auto，所以如果不指定left和right
                            则等式不满足时，会自动调整这两个值

 垂直方向布局的等式的也必须要满足
   bottom + top + margin-top/bottom + padding-top/bottom + border-top/bottom + height = 包含块的高度

 .demo1{
            width: 300px;
            height: 300px;
            background-color: #bfa;
            position: relative;
            /* 4.demo1开启相对定位 因此demo2的包含块更改为demo1所在的文档流位置 */
        }
        .demo2{
            width: 100px;
            height: 100px;
            background-color: skyblue;
            /* 1.此时demo1 和 demo2 上边距重叠 */
            position: absolute;
            /* 2.demo2开启绝对定位 */
            left: 0;
            top: 0;
            /* 3.demo2回到定位参照原点 此时其父元素还没开启定位 所以其包含块应该是html根元素*/
            right: 0;
            /* 5.设置 right == 0 但是水平的等式 过度约束仍然成立 此时浏览器自动调整right不为0以满足等式 */
            margin-left: auto;
            /* 6.设置demo2 的margin-left为auto 此时等式不成立是 浏览器优先调整margin-left为最大值以满足等式 */
            margin-right: auto;
            /* 7.此时demo2 的margin-right 也为auto 此时浏览器自动调整水平两边的margin值相等 达到居中 */
            bottom: 0;
            margin: auto;
            /* 8.此时demo2的水平 垂直方向上margin 均为 auto 所以浏览器优先调整margin 达到水平垂直居中 */
        }
 <div class="demo1">
        <div class="demo2"></div>
    </div>


PS： 在绝对定位中 若定位元素未设置宽高时 且此时的margin == auto（四个方向），right=left=top=bottom=0 时 此时浏览器自动优先调整元素宽高为父元素大小以满足等式
	即margin与宽高均为auto时（元素开启绝对定位后 脱离文档流 元素的宽高默认被内容撑开 可看作为auto）此时宽高将被调整成最大值

7.元素的层级

对于开启了定位元素，可以通过z-index属性来指定元素的层级
                    z-index需要一个整数作为参数，值越大元素的层级越高
                        元素的层级越高越优先显示

                    如果元素的层级一样，则优先显示靠下的元素

                    祖先的元素的层级再高也不会盖住后代元素

 .demo1{
            width: 200px;
            height: 200px;
            background-color: #bfa;
            position: relative;
            /* 1.开启相对定位 */
            z-index: 1;
            /* 11.给demo1设置层级 此时demo1层级变为最高 */
        }
        .demo2{
            width: 200px;
            height: 200px;
            background-color: skyblue;
            position: absolute;
            /* 2.开启绝对定位 */
            top: 20px;
            left: 40px;
            /* 5. 此时移动demo2的位置 此时demo2 的参照原点是html根元素 可知demo2 的层级高于demo1 */
        }
        .demo3{
            width: 200px;
            height: 200px;
            background-color: gold;
            /* 3.此时demo3移动到demo2下面 因为demo2已脱离文档流 且demo2此时的层级高于demo3*/
            position: absolute;
            /* 4.demo3也开启绝对定位 越靠后的元素层级越高 所以此时demo3层级高于demo2 */
            left: 80px;
            top: 40px;
            /* 6. 此时移动demo3的位置 此时demo3 的参照原点是html根元素 可知demo3 的层级高于demo2 */
            z-index: 2;
            /* 12.给demo3设置层级 此时demo3层级高于demo1 demo3的层级此时最高 但可知祖先的层级再高也不会遮挡后代元素 */
        }
        .demo4{
            width: 100px;
            height: 100px;
            background-color: green;
            /* 7.demo4 完全被包含在demo3中 */
            position: absolute;
            /* 8.demo4开启绝对定位 还没设置偏移量 所以位置不变 且demo4已没有元素 */
            top: 0;
            left: 0;
            /* 9.此时demo4回到定位参照原点 可知此时demo4的包含块是demo3 */
            top: 120px;
            left: 140px;
            /* 10.移动 demo4的位置 可知此时demo4 的层级高于demo3 此时层级为 demo1 < demo2 < demo3 < demo4 */
        }
 <div class="demo1">1</div>
    <div class="demo2">2</div>
    <div class="demo3">3
        <div class="demo4">4</div>
    </div>
