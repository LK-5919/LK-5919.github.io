Element-ui组件的使用

2022-07-17 开发日记 3.5k 字
WEB前端

题记

此文用于记录在使用Element-ui中遇到的一下问题。

文件上传

在开发中有使用到el-upload组件进行文件上传，在这其中在上传图片时，小图片采取的是通过将本地图片转为Base64格式再上传到后端数据库进行存储，还有另外一种是直接将图片以file的格式上传到后端服务器，服务器储存文件成功后向前端返回图片存储的在线地址，之后提交表单时再将这个在线地址一起提交保存到服务器。在这里先来讲述一下el-upload组件的简单使用；

<el-col :span="12">
         <el-form-item label="器械图标" prop="machinePicBase64">
           <el-upload
             action="#"
             :headers="myHeaders"
             multiple="false"
             list-type="picture-card"
             accept=".jpg, .png"
             :http-request="imageToBase64"
             :before-upload="beforeUpload"
             :limit="1"
             :file-list="fileList"
             :on-remove="handleRemove"
             :class="{ disabled: uploadDisabled }"
             :on-success="handleAvatarSuccess"
           >
             <i class="el-icon-plus"></i>
           </el-upload>
         </el-form-item>
       </el-col>

el-upload组件的一些参数使用：

  • list-type: 为文件列表的类型，string — text/picture/picture-card；
  • accept: 接受上传的文件类型；
  • limit: 最大允许上传个数;
  • multiple: 是否支持多选文件;
  • before-upload: 上传文件之前的钩子，参数为上传的文件，若返回 false 或者返回 Promise 且被 reject，则停止上传；
  • file-list: 上传的文件列表，数组类型；
  • action：必选参数，为文件上传的后端地址，当使用默认上传行为的时候将走此后端地址；
  • headers: 设置默认上传的请求头部，为Object类型；
  • http-request: 值定义自定义上传行为，此值将覆盖默认上传方法；
  • on-remove: 文件列表移除文件时的钩子，默认传入file和fileList参数；
  • on-success: 文件上传成功时的钩子，默认传入response, file, fileList参数；

在上传文件的时候我们可以判断文件的类型和文件的大小，在这里已经使用了accept来校验上传的文件类型，所以我们可以在before-upload钩子函数中部检验文件的大小，方法如下：

 //上传图片前的钩子--检验图片大小
//此钩子函数默认传入file参数
    beforeUpload(file) {
      const isLt2M = file.size / 1024 / 1024 < 2;
      if (!isLt2M) {
        this.$message.error('上传图片大小不能超过 2MB!');
        return false;
      }else{
        //当文件大小符合大小时，可往fileList数组中放入此上传的文件；（此操作不是必要）
        this.fileList.push(file.row);
        return isLt2M;
      }
    },

在上传小图片转为base64存储时，不需要上传到服务器，此时就可以不使用默认上传方法，此时action='#'，在http-request='imageToBase64'定义自定义上传方法，其中imageToBase64为在method中自己定义的方法。本地图片转base64的方法如下：

 // 自定义上传方法 处理图片转化为base64
//默认传入file参数
    imageToBase64(file) {
      let reader = new FileReader();
      reader.readAsDataURL(file.file);
      reader.onload = function (e) {
        //去除前缀--base64有个统一的前缀 'data:image/png;base64,'
        //在这里只截取保存逗号后面的字符串
        let img_base64 = e.target.result.split(',')[1];
        this.dialogData.machinePicBase64 = img_base64;
      }
    },

在上传大图片的时候，可采用默认上传行为:action='IP'，在这里IP为在data中自定义的地址接口，在这里有个必须要注意的点，在项目中，我们的每次请求一般都会在请求头中添加token后再进行发送，在我的这个项目中也是用到了axios的请求拦截器统一进行请求头添加token，但是我发现el-upload的默认上传方法并不是走我封装好的axios，也就是说默认上传方法并不会主动添加token请求头，这就需要我们在:headers中配置默认上传方法需要携带的请求头，代码如下：

data() {
       return {
           fileList: [],  //文件上传列表
           IP: '/api/upload', //自己后端的文件上传地址
           myHeaders: { 
               token: sessionStorage.getItem("token"), 
                       },
       }
   },
methods: {
   //上传教师头像成功的回调
   handleAvatarSuccess(res, file) {
     if (res.code == '0') {
         //将服务器返回的在线地址赋给表单数据
       this.dialogData.avatarUrl = res.data
     }
   },
  // 移除图片回调--清空fileList
   handleRemove(file, fileList) {
     this.fileList = []
   },
}

文件上传 — 限制只上传一个文件

项目中有一个需求是需要在上传一个文件成功后隐藏上传按钮，在此我才用判断fileList数组当前是否为空，当数组长度不为0时，通过css样式将上传按钮隐藏，当移除已经上传 的图片即清空fileList时再渲染上传按钮，所以需要给el-upload组件绑定一个动态类:class='{ disabled: uploadDisabled }'，通过计算属性及时判断当前的fileList数组长度来返回disabled的值，代码如下：

computed: {
    uploadDisabled: function () {
      return this.fileList.length > 0
    },
  },

其中动态类样式为：

//在这里因为我的el-upload组件再dialog的表单中，故需要添加 /deep/ 使样式生效
/deep/ .disabled {
  .el-upload--picture-card {
    display: none;
  }
}

文件上传 — 图片回显

在项目中，有一个需求是当点击表单编辑时，需要再dialog中表单的el-upload组件回显之前上传成功的图片，在这里可以给el-upload组件的fileList中放入图片的在线url或者base64编码进行图片的回显，代码如下：

// 编辑表单方法
//data为传入编辑表单的数据
    modify(data) {
      //编辑时图片回显
      this.fileList.push({
        url: 'data:image/jpg;base64,' + data.icon, //这里也可以是图片的在线地址
      });
      //其他数据操作，在此省略
      ...
      this.dialog.visible = true
    },

不过其中展开的是表格结构，图中的是表单结构，利用的是element-ui中提供的type="expand"；在el-table-column 组件中开启上面那个type属性，之后利用插槽slot-scope="props"，此时props中为当前行数据，之后展开的子表格中需要的数据即从props中获取；

在这里复习一下Vue中三种插槽的简单使用方法：

  • 默认插槽：又名匿名插槽，不用设置名称属性，单个插槽可以放置在组件的任意位置，一个组件只能有一个默认插槽。
//子组将中
<template>
    <div>
        <h2>子组件 插槽使用</h2>
	//在子组件中为插槽预留位置
        <slot></slot>    
	
        //默认内容，插槽还可以使用默认内容，在不传递值时显示默认内容的值
	<solt>默认内容</solt> 
    </div>
</template>


//父组将中使用
import Slotdemo from "./soltDemo.vue";
components: {Slotdemo},
    
<template>
    <div>
        <!-- 插槽使用 -->
        <Slotdemo>
            <div class="solt">
                <div>
                    <el-input placeholder="请输入内容"></el-input>
                </div>
            </div>
        </Slotdemo>
    </div>
</template>
  • 具名插槽：当子组件功能复杂时，子组件的插槽可能并非一个，这个时候就需要给slot指定一个name属性，也就是具名插槽。
//子组件中
<template>
    <div>
        <!-- 具名插槽 -->
        <slot name="title">
            title具名插槽默认内容
        </slot>
        <div>
            <slot name="count">
                count具名插槽默认内容
            </slot>
        </div>
    </div>
</template>


//父组件中使用
         import Slotdemo from "./soltDemo.vue";
	 components: {Slotdemo},
        <!-- 具名插槽使用 -->
        <!-- 注意：具名插槽一定要在外面包裹一层 template 并在其上面使用  v-solt=" " 属性来指定插槽名称-->
        <Slotdemo>
             
            <template v-solt="title">     
                <div>title具名插槽 向名字为title的插槽插入内容</div>
            </template>

            <template v-solt="count">
                <div>count具名插槽 向名字为count的插槽插入内容</div>
            </template>
        </Slotdemo>
  • 作用域插槽：我们可以将其叫做带数据的插槽，父组件在使用的时候可以替换slot插槽中的显示页面结构，但展示的数据还是来源于子组件。当数据在子组件中，但是插槽中渲染数据的节点由父组件决定，这种情况下，就需要使用作用域插槽。
//子组件中使用 solt 插槽将数据 传递至父组件

<template>
    <div>
        <slot :list="list"></slot>
    </div>
</template>
<script>
export default {
    data() {
        return {
            list: ['张三', '李四', '王五']
        }
    },
}
</script>


//父组件中使用作用域插槽
	import Slotdemo from "./soltDemo.vue";
	components: {Slotdemo},
        
        <!-- 作用域插槽 -->
        <!-- 作用域插槽的使用必须要在子组件标签上面使用v-slo 获取到作用域插槽在子组件中的值再在后续中使用 -->
        <Slotdemo v-slot="{ list }">
            <!-- 作用域插槽 -->
            <ol>
                <li v-for="item in list" :key="item.id">
                    {{item.count}}
                </li>
            </ol>
        </Slotdemo>

支持远程搜索的下拉框

在项目中，有一个需求是需要实现用户在文本输入框中键入关键字后，页面自动将关键字发送至后台进行检索，并将检索的查询结果作为下拉框的options进行渲染，具体代码如下：

<el-select
             v-model="dialogData.courseIdList"
             multiple
             collapse-tags
             filterable
             remote
             reserve-keyword
             placeholder="请输入课程关键词"
             clearable
             :remote-method="remoteMethod"
             :loading="loading"
           >
             <el-option
               v-for="item in courseOptions"
               :key="item.value"
               :label="item.label"
               :value="item.value"
             >
             </el-option>
           </el-select>

为了启用远程搜索，需要将filterable和remote设置为true，同时传入一个remote-method。remote-method为一个Function，它会在输入值发生变化时调用，参数为当前输入值。需要注意的是，如果el-option是通过v-for指令渲染出来的，此时需要为el-option添加key属性，且其值需具有唯一性，比如此例中的item.value。mutiple为开启下拉框多选；

远程异步搜索方法如下：

//异步远端搜索，query为默认传入的关键字
    remoteMethod(query) {
      try {
        clearTimeout(a)
      } catch (e) {}
      var a = setTimeout(() => {
        this.loading = true
        if (query !== '') {
          getCourse({ courseName: query }).then((res) => {
            if (res.data.code === '0') {
                //remoteOptions数组存放后端返回的数组
              this.remoteOptions = res.data.data.map((item) => {
                return { value: item.value, label: item.label }
              })
              this.loading = false
            }
          })
        } else {
          this.loading = false
          this.remoteOptions = []
        }
      }, 300)
    },

在实现这个功能需求后，还涉及到需要在编辑表单回显数据到此下拉框，故在这里采取了使用两个数组存放数据：

data() {
    return {
      loading: false,
      remoteOptions: [], //远程模糊搜索返回的数据
      currentOptions: [],  //用于修改时回显的数据
},
computed:{
    courseOptions(){
        //合并数组
        let options = JSON.parse(JSON.stringify(this.remoteOptions.concat(this.currentOptions)));
        //数组去重
        return [...new Set(options)];
    },
  }

在编辑表单时，currentOptions存放后端数据库目前已保存的id，remoteOptions存放远程搜索的结果，el-option组件遍历的是计算属性将currentOptions和remoteOptions两个数组合并并去重的结果；

Vue中数据更新不及时的问题

当给Vue的一个对象在method方法中动态添加属性时，会存在添加的数据不会立刻更新到页面的问题，因为我们此时添加的属性并不是响应式数据，不能被Vue观察到，可用如下方法给Vue的data中的对象添加属性：

 //添加响应式数据，确保数据及时更新到视，userForm为Object类型，sumFare为醉驾到其身上的新属性
//data[0].sumFare为属性值
Vue.set(this.userForm,'sumFare',data[0].sumFare);
